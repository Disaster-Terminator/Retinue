# Retinue v0.1.0 发布说明

Retinue v0.1.0 是第一个可以日常使用的版本：Codex 安装插件后，可以把 OpenCode 当作本地子代理来调用，完成 spawn、wait、list、close 的基本流程。

英文发布说明见 [v0.1.0_RELEASE_NOTES.md](v0.1.0_RELEASE_NOTES.md)。

## 这一版能做什么

- 通过 Codex 插件市场安装；用户不需要克隆仓库或本地编译。
- 默认使用 OpenCode 后端，并让 OpenCode 使用 `plan` agent。
- Retinue 默认管理本机 OpenCode server 生命周期。
- 如果 `4096` 被外部服务占用，Retinue 会继续尝试 `4097` 到 `4127`。
- Codex 可以使用 4 个产品工具：
  - `retinue_spawn_agent`
  - `retinue_wait_agent`
  - `retinue_close_agent`
  - `retinue_list_agents`
- 每个 MCP 会话都有独立的子代理槽位。超过上限时，Retinue 会关闭最早仍在运行的子代理，给新任务腾出位置。
- `retinue_wait_agent` 在任务结束时返回子代理结果；任务仍在运行时，返回 job 路径和受限的 stdout/stderr 尾部输出，方便主代理判断是继续等、看日志，还是关闭任务。OpenCode 出现空 assistant 或未完成 assistant 输出并超过诊断阈值后会进入 `stalled`，不会被当成成功结果。
- 发布流程已经加入 package smoke，确保插件包里的 MCP runtime 能真正启动，而不是只在源码目录里可用。

## 安装方式

先添加插件市场：

```bash
codex plugin marketplace add Disaster-Terminator/Retinue
```

然后在 Codex 里运行 `/plugins`，切到 `[Retinue Local]`，进入 `Retinue` 详情页，选择 `Install plugin`。安装后重启 Codex。

重启后可以直接让 Codex 做一次冒烟测试：

```text
Use Retinue to spawn an OpenCode plan subagent. Ask it to reply exactly: RETINUE_OK. Wait for the result and close the child agent.
```

## 运行要求

- Node.js 20+
- 支持插件的 Codex CLI
- OpenCode 1.14+，推荐使用官方安装脚本：

```bash
curl -fsSL https://opencode.ai/install | bash
```

## 边界

- OpenCode 继续负责 provider、model、登录状态、额度和 profile 配置。
- Retinue 不在每次 tool call 参数里选择 backend/profile/model/agent/permission。
- Claude Code 后端保留给自定义部署，但 v0.1.0 的默认产品路径是 OpenCode。

## v0.1.0 之后的加固

初版 v0.1.0 基线之后，Retinue 增加了 OpenCode soft-stall recovery。可恢复的无最终文本卡住会在当前 wait 超时时间内继续等待，并向同一个子会话提交一次 no-tools final-answer recovery prompt。默认 MCP wait 上限是 180 秒，让默认 75 秒 soft-stall 检测有足够时间完成这次 rescue。provider error 和只读任务里的 patch/write intent 仍然会立即返回 `stalled`。
