# Retinue 0.1.0 Hardening Record

Opened: 2026-05-07

This document records issues found during real Codex plugin smoke and E2E work. Unless explicitly marked as follow-up work, the issues below are closed within the v0.1.0 release boundary.

## Release Boundary

v0.1.0 default product path:

- Codex plugin marketplace install.
- Default backend: OpenCode.
- Default OpenCode agent: `explore`.
- Claude Code remains experimental and optional.
- New users should not need to build Retinue or hand-edit backend configuration.

## Closed Issues

### 1. MCP Startup From The Plugin Marketplace Cache

Closed.

Retinue now ships a complete plugin runtime and starts through `plugins/retinue/mcp-bootstrap.mjs`. The bootstrap loads plugin-local `dist/mcp.js` while moving the long-lived process cwd out of the plugin cache directory.

Guardrails:

- CI covers MCP startup from an isolated plugin cache.
- `pnpm run smoke:package` starts `dist/mcp.js` and plugin-local `mcp-bootstrap.mjs`.
- `pnpm run check:generated` fails when generated runtime is not committed.

### 2. Default OpenCode Server Lifecycle

Closed.

The plugin defaults enable OpenCode auto-serve:

- `RETINUE_OPENCODE_AUTO_SERVE=1`
- `RETINUE_OPENCODE_HOST=127.0.0.1`
- `RETINUE_OPENCODE_AGENT=explore`

### 3. Windows / WSL Port Conflicts

Closed.

Retinue-managed auto-serve prefers `4096`; when earlier ports are occupied by external services, it tries later ports through `4127`. Explicit port configuration remains strict and does not silently move to another port.

### 4. Multiple Codex Threads Using Retinue

Closed.

Retinue writes managed OpenCode server discovery into the state directory, protects startup with a cross-process lock, reuses healthy managed servers, and avoids attaching to external OpenCode servers that Retinue does not own.

### 5. Empty Results From Real OpenCode Plan Subagents

Closed.

Retinue no longer treats assistant messages without visible text as successful results. Long tool-call/no-text loops, empty `finish=stop` assistant rounds, and incomplete assistant tool rounds become `stalled` with diagnostics instead of leaving the caller with an indefinite no-output wait. While running, `retinue_wait_agent` returns job paths and bounded stdout/stderr tails.

### 6. Plugin Cache Refresh And Locked Cache Entries

Closed for Retinue packaging.

Retinue avoids holding the plugin cache as a long-lived cwd, ships artifact smoke checks, and provides `dev:sync-plugin-cache` for fast local development reloads. Codex host cache-lock or cache-refresh failures remain host behavior to diagnose if observed.

### 7. OpenCode Auto-Serve Spawn Errors

Closed.

OpenCode startup now observes health polling, child `error`, and early `exit`. Retinue resolves official-script install locations and common package-manager shims. Startup failures return readable tool errors instead of closing the MCP transport.

## Current Verification Requirements

Before tagging or publishing a release, run:

```bash
pnpm run typecheck
pnpm test
pnpm run check:generated
pnpm run smoke:package
pnpm run verify:package
RETINUE_REAL_OPENCODE_SLOT_PROBE=1 RETINUE_MAX_CONCURRENT_AGENTS=1 pnpm run probe:real:retinue-opencode-slots
```

- Add richer Retinue configuration after the v0.1.0 plugin path is released.
- Continue Windows/WSL plugin cache diagnostics if Codex reports locked or stale plugin cache entries.
