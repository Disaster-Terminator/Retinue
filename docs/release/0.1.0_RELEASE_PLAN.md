# Retinue 0.1.0 Release Readiness

This document records the final state before the v0.1.0 release. It is no longer an active work plan; follow-up work should go into later release notes or issues.

Chinese release notes are available at [v0.1.0_RELEASE_NOTES.zh-CN.md](v0.1.0_RELEASE_NOTES.zh-CN.md).

## Release Boundary

Retinue 0.1.0 is the first usable local-first Codex subagent plugin release.

The release boundary is:

- Plugin marketplace installation is the default user path.
- OpenCode is the default backend.
- The default OpenCode agent is `plan`.
- Codex can see the Retinue skill and product MCP tools.
- A user can ask Codex to create an OpenCode-backed child agent, wait for the result, list active child agents, and close the child agent.
- Retinue manages the default local OpenCode server lifecycle. If earlier ports in `4096` through `4127` are occupied by external services, Retinue tries later ports in that range.
- The npm package remains available for custom MCP deployments.

## Released Tool Surface

Retinue 0.1.0 exposes these product tools:

- `retinue_spawn_agent`
- `retinue_wait_agent`
- `retinue_close_agent`
- `retinue_list_agents`

These tools do not accept backend, profile, model, agent, permission mode, or OpenCode base URL as per-call arguments. Those choices belong to deployment configuration, not individual tasks.

`retinue_wait_agent` returns the child result when the child reaches a terminal state. While the child is still running, it returns the state directory, trace path, job artifact paths, and bounded stdout/stderr tails so the caller can decide whether to wait, inspect logs, or close the task. OpenCode empty-output assistant rounds, incomplete assistant tool rounds, and long no-text tool-call loops are reported as `stalled` with diagnostics only after conservative thresholds are crossed. A single MCP wait timeout is a polling event, not a failed child-agent result.

## Closed Before Tag

- The quick start uses plugin marketplace installation and Codex `/plugins`.
- Source build instructions are outside the quick start.
- `dist/` and `plugins/retinue/dist/` are committed so plugin installs do not require users to build locally.
- The npm package name is `@disaster-terminator/retinue`.
- CLI bins include `retinue`, `retinue-mcp`, `retinued`, and `retinue-daemon`.
- Plugin MCP defaults are:
  - `RETINUE_BACKEND=opencode`
  - `RETINUE_OPENCODE_AUTO_SERVE=1`
  - `RETINUE_OPENCODE_HOST=127.0.0.1`
  - `RETINUE_OPENCODE_AGENT=plan`
- Plugin marketplace metadata marks Retinue as installable.
- Package verification checks docs, plugin files, and runtime files.
- Package smoke starts `dist/cli.js`, `dist/mcp.js`, and plugin-local `mcp-bootstrap.mjs`.
- Local development has passed real OpenCode E2E and multi-agent slot probes.

## User Acceptance Smoke

Add the plugin marketplace:

```bash
codex plugin marketplace add Disaster-Terminator/Retinue
```

Then open Codex, run `/plugins`, select `[Retinue Local]`, open the `Retinue` details page, choose `Install plugin`, and restart Codex.

After restart, ask Codex:

```text
Use Retinue to spawn an OpenCode plan subagent. Ask it to reply exactly: RETINUE_OK. Wait for the result and close the child agent.
```

Acceptance points:

- The Retinue skill is visible.
- `retinue_spawn_agent` is callable.
- `retinue_wait_agent` returns a result containing `RETINUE_OK`.
- `retinue_close_agent` returns a terminal status.
- `retinue_list_agents` can show active child agents in the current MCP session.

## Follow-Up Work

These items are post-v0.1.0 work and do not block this release:

- Add a Retinue config file to reduce reliance on environment variables.
- Decide whether OpenCode `build` agent support belongs in 0.1.x as deployment-level configuration.
- Document reproducible Claude Code profile requirements before treating Claude Code as a first-class default path.
- Add `retinue_send_message` and `retinue_followup_task` after continuation semantics are stable.
- Continue Windows/WSL plugin cache diagnostics if Codex reports locked or stale plugin cache entries.
